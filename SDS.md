# 🧭 System Description Specification (SDS)

## Númer teymis og höfundar

## Heiti kerfis
Stoð

## Hvað er kerfið?
Stoð er heildstætt stafrænt umönnunarkerfi hannað til að styðja við þjónustu fyrir fatlað fólk og aðra þjónustunotendur í umönnun. Kerfið er fyrir starfsfólk í umönnunargeiranum, þjónustunotendur sjálfa, aðstandendur þeirra og stjórnendur þjónustunnar.
## Tilgangur
Meginmarkmið kerfisins er að auka lífsgæði þjónustunotenda og skilvirkni starfsfólks með því að einfalda daglegt skipulag, bæta yfirsýn yfir lyfjagjafir og verkefni, tryggja samfellu í þjónustu og veita gagnvirk stafræn stuðningsverkfæri.

## Afmörkun (Scope)
**Innan scope:**

- Aðgengilegar upplýsingar um sértækar þarfir, greiningar og stuðningsáætlanir þjónustunotenda.

- Dagbókarskráning á líðan og atburðum.

- Verkefnastjórnun fyrir starfsfólk (dagleg verk, vaktaskipulag).

- Utanumhald og áminningakerfi fyrir lyfjagjafir.

- Grunnvirkni í samskiptum milli starfsfólks og við þjónustunotendur.

- Einfalt spjallmenni (AI) sem veitir aðstoð við algengum spurningum fyrir bæði starfsfólk og þjónustunotendur.

**Utan scope:** 

- Kerfið sér ekki um launavinnslu eða bókhald.

- Kerfið heldur ekki utan um læknisfræðilegar sjúkraskrár, aðeins daglegar umönnunarupplýsingar.

- Kerfið er ekki ætlað fyrir neyðarviðbrögð í rauntíma (eins og 112 þjónusta).

## Ytri tenging 
- Rafræn skilríki: Kerfið gæti í framtíðinni tengst island.is fyrir örugga auðkenningu aðstandenda og stjórnenda.

## Tenging við SRS
- Sjá nánari kröfuskipan í `SRS.md` (viðskiptakröfur, fídusar, notendakröfur o.s.frv.).

## Verkaskipting
- Hugmyndavinna og kröfugreining: Unnum saman í hóp á fundum til að skilgreina helstu notendahópa og markmið kerfisins.

- Skráning krafna: Við skiptum með okkur flokkum krafna til að skrá sem Issues í GitHub.

- Rýni: Hver meðlimur rýndi Issues hjá hinum á Kanban og gaf athugasemdir áður en þau voru samþykkt.

- Git-flæði: Vinnuflæðið í Git fór þannig fram að allir meðlimir unnu beint á main branch og sendu (push) breytingar jafnóðum. Samskipti um framvindu og breytingar fóru fram munnlega og með yfirsýn á Kanban borðinu, en ekki í gegnum formlegt rýniferli í GitHub.
  
## Ígrundun 

 ### Hvað gekk vel í verkefninu 
- Samvinnan í hópnum var mjög góð og allir voru virkir þátttakendur.

- Notkun á GitHub Projects (Kanban) gaf okkur frábæra yfirsýn yfir stöðu verkefnisins og hvað þyrfti að gera næst.

- Verkaskiptingin var skýr sem gerði vinnuna markvissa og skilvirka.
 ### Hvað má bæta næst 
- Við hefðum getað verið nákvæmari í lýsingum á sumum Issues frá byrjun til að forðast misskilning.

- Við sjáum að við hefðum getað nýtt Git mun betur. Í næsta verkefni ætlum við að tileinka okkur að vinna í aðskildum greinum (branches) og nota Pull Requests til að rýna vinnu hvers annars. Þannig getum við komið í veg fyrir árekstra og tryggt meiri gæði áður en breytingar eru sameinaðar við main branch.
